Phase 1 — Buy Domain at Cloudflare Registrar

  1. Go to Cloudflare Dashboard → dash.cloudflare.com → your account
  2. Add site → enter apsmono.com
  3. Registrar tab → Register domain → search apsmono.com
  4. Purchase — should be ~$10.44/year. No markup.
  5. DNS is automatic — Cloudflare manages it. No external nameserver changes.
🛡️ Optional: Buy .dev Too

If you want apsmono.dev as a redirect: buy it in the same account (~$12-15/year). We'll 301 it to apsmono.com later in Phase 2.

🔴 Keep Proxy Orange

Always keep the cloud orange (Proxied). This gives you Cloudflare SSL, DDoS protection, and edge caching. Gray (DNS-only) exposes your origin IP.

For apsmono.dev redirect

Add these records, then create a Page Rule:

CNAME @ apsmono.com (Proxied)
CNAME www apsmono.com (Proxied)

# Rules → Page Rules
URL: apsmono.dev/*
Setting: Forwarding URL → 301 → https://apsmono.com/$1

Phase 3 — GitHub Pages Custom Domain

3a — Root User Site (apsmono.github.io repo)

  1. Repo settings → Pages → Custom domain → enter apsmono.com
  2. Enforce HTTPS → check the box (after DNS propagates)
  3. Wait for green checkmark — DNS verification takes ~1 hour
📄 Root Site Index

If the root repo is empty, create a minimal index.html that redirects to /dashboard/:

<!DOCTYPE html>
<html>
<head>
  <meta http-equiv="refresh" content="0; url=/dashboard/">
</head>
<body>
  <p>Redirecting to <a href="/dashboard/">dashboard</a>...</p>
</body>
</html>

3b — Dashboard (apsmono/dashboard)

  • Repo settings → Pages → Custom domain → LEAVE BLANK
  • Keep base: '/dashboard/' in vite.config.ts
  • Because root site has custom domain, project pages auto-inherit it
  • Result: https://apsmono.com/dashboard/

3c — Wedding Invitation (apsmono/wedding-invitation)

  • Repo settings → Pages → Custom domain → LEAVE BLANK
  • Keep base: '/wedding-invitation/' in vite.config.ts
  • Result: https://apsmono.com/wedding-invitation/
💡 Why Subpaths, Not Subdomains?

Subpaths require zero extra DNS. Subdomains need separate CNAMEs and each repo needs custom domain config. GitHub Pages' inheritance is the path of least resistance.

Phase 4 — Cloudflare Pages: Koperasi Custom Domain

  1. Cloudflare Dashboard → Pages → koperasi project
  2. Custom domains → Set up a custom domain
  3. Enter koperasi.apsmono.com
  4. Cloudflare auto-creates the DNS record (confirms Phase 2 record)
  5. Wait for SSL provisioning — usually < 5 minutes

Phase 5 — Cloudflare Tunnel (MacMini Backend)

This is the magic. No port forwarding. No DuckDNS. No certbot.

5a — Install cloudflared

# On MacMini
brew install cloudflared
cloudflared --version

5b — Authenticate

cloudflared tunnel login

Opens browser → select your Cloudflare account → authorize. Certificate downloads to ~/.cloudflared/cert.pem.

5c — Create Tunnel

cloudflared tunnel create api-apsmono
# Save the UUID output

5d — Configure

Create ~/.cloudflared/config.yml:

tunnel: <UUID-from-step-5c>
credentials-file: /Users/<you>/.cloudflared/<UUID>.json

ingress:
  - hostname: api.apsmono.com
    service: http://localhost:8000
    originRequest:
      noTLSVerify: true
  - service: http_status:404
⚠️ localhost:8000

This assumes Docker Compose maps host port 8000 to the container. If your docker-compose.yml uses a different host port, change this.

5e — Route DNS

cloudflared tunnel route dns api-apsmono api.apsmono.com

5f — Run as Service

sudo cloudflared service install
sudo cp ~/.cloudflared/config.yml /etc/cloudflared/config.yml
sudo cp ~/.cloudflared/<UUID>.json /etc/cloudflared/
sudo launchctl load /Library/LaunchDaemons/com.cloudflare.cloudflared.plist

Verify:

curl https://api.apsmono.com/healthz

Phase 7 — Solo-leveling Backend Updates

7a — Update .env

# solo-leveling/.env
FRONTEND_ORIGIN=https://apsmono.com
API_BASE=https://api.apsmono.com

7b — Update CORS

In your FastAPI app (src/app.py or similar):

origins = [
    "https://apsmono.com",
    "https://www.apsmono.com",
    "http://localhost:5173",
    "http://localhost:8080",
    "http://macmini.local:8000",
]

app.add_middleware(
    CORSMiddleware,
    allow_origins=origins,
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)
🔴 Credentials + Wildcards

When allow_credentials=True, you cannot use allow_origins=["*"]. FastAPI rejects this combo. List every origin explicitly.

7c — Restart

cd solo-leveling
docker compose down
docker compose up -d
