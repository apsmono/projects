Pattern 2 (intent_parser.py):
```python
_INTENT_SYSTEM_PROMPT = """You are an intent parser for a personal knowledge system.
Given a user's natural language command, classify it into one of these intents and extract parameters.

Intents: library_search, library_capture, library_qa, status, help, park_distraction, unknown

Respond with valid JSON only:
{
  "intent": "library_search",
  "params": {"query": "machine learning papers"},
  "confidence": 0.95
}"""

async def parse_intent(text: str) -> dict[str, Any]:
    try:
        response = run_agent(
            task=f"Parse this command: {text}",
            system=_INTENT_SYSTEM_PROMPT,
        )
        parsed = json.loads(response)
        return {
            "intent": parsed.get("intent", "unknown"),
            "params": parsed.get("params", {}),
            "confidence": parsed.get("confidence", 0.5),
        }
    except Exception:
        from src.core.router import _detect_intent
        intent = _detect_intent(text)
        return {"intent": intent, "params": {"text": text}, "confidence": 0.3}
```

Router evolution (from RESEARCH.md):
```python
async def route_command_async(text: str, source: str = "api") -> str:
    from src.core.intent_parser import parse_intent
    parsed = await parse_intent(text)
    intent = parsed["intent"]
    params = parsed.get("params", {})
    logger.info("Intent detected: %s (confidence=%.2f)", intent, parsed.get("confidence", 0))
    reply = await _dispatch_async(intent, text, params)
    _log_command(text, intent, reply, source)
    return reply
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User text -> Intent parser | Untrusted natural language crosses into LLM prompt |
| LLM response -> JSON parser | Untrusted LLM output crosses into json.loads |
| Intent -> Dispatch | Parsed intent determines which handler runs |
| Guide API -> Library store | Park endpoint writes to filesystem |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-09 | Spoofing | Prompt injection in intent parser | mitigate | System prompt hardcodes allowed intents; params are validated before dispatch; no arbitrary code execution |
| T-03-10 | Tampering | Malformed LLM JSON crashes parser | mitigate | try/except with fallback to keyword map; json.loads wrapped in exception handler |
| T-03-11 | Denial of Service | LLM call blocks event loop | accept | run_agent is ~200ms HTTP call; acceptable for personal use; if latency becomes issue, add debounce in UI |
| T-03-12 | Elevation of Privilege | Park endpoint without auth | mitigate | All guide endpoints use Depends(require_auth) |
| T-03-13 | Information Disclosure | Metrics leak other user data | mitigate | Metrics computed from library store scoped by owner_id; single-tenant today |
| T-03-14 | Denial of Service | Park endpoint spam | mitigate | Same auth + rate limiting as other endpoints; no special exposure |
</threat_model>
